This package provides access to \href{https://dxfeed.com/market-data/}{\texttt{ dx\+Feed market data}}. The library is built as a language-\/specific wrapper over the \href{https://dxfeed.jfrog.io/artifactory/maven-open/com/dxfeed/graal-native-api/}{\texttt{ dx\+Feed Graal Native}} library, which was compiled with \href{https://www.graalvm.org/latest/reference-manual/native-image/}{\texttt{ Graal\+VM Native Image}} and \href{https://docs.dxfeed.com/dxfeed/api/overview-summary.html}{\texttt{ dx\+Feed Java API}} (our flagman API).

\+:information\+\_\+source\+: If you already use \href{https://github.com/dxFeed/dxfeed-c-api}{\texttt{ dx\+Feed C API}}, please see the Overview section.~\newline
 \+:warning\+: It’s an {\bfseries{alpha}} version and still under active development. {\bfseries{Don’t use it in a production environment.}}

\hypertarget{index_autotoc_md9}{}\doxyparagraph{\texorpdfstring{Implementation Details}{Implementation Details}}\label{index_autotoc_md9}
We use \href{https://www.graalvm.org/latest/reference-manual/native-image/}{\texttt{ Graal\+VM Native Image}} technology and specially written code that {\itshape wraps} Java methods into native ones to get dynamically linked libraries for different platforms (Linux, mac\+OS, and Windows) based on the \href{https://dxfeed.jfrog.io/artifactory/maven-open/com/devexperts/qd/dxfeed-api/}{\texttt{ latest Java API package}}.

Then, the resulting dynamic link library (dx\+Feed Graal-\/native) is used through C \href{https://en.wikipedia.org/wiki/Application_binary_interface}{\texttt{ ABI}} (application binary interface), and we write programming interfaces that describe our business model (similar to Java API).

As a result, we get a full-\/featured, similar performance as with Java API. Regardless of the language, writing the final application logic using API calls will be very similar (only the syntax will be amended, \texorpdfstring{$\ast$}{*}\"{}best practices\"{}\texorpdfstring{$\ast$}{*}, specific language restrictions)
